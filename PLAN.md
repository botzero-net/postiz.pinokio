# Postiz Pinokio Launcher - Implementation Plan

## Overview

Build a 1-click Pinokio launcher for [Postiz](https://github.com/gitroomhq/postiz-app) - an open-source social media scheduling tool with AI features.

**Repo:** https://github.com/botzero-net/postiz.pinokio

---

## Configuration Strategy

### Tier 1: Required (Auto-generated)
These are generated on first install with secure defaults:

```env
MAIN_URL=http://localhost:4007
FRONTEND_URL=http://localhost:4007
NEXT_PUBLIC_BACKEND_URL=http://localhost:4007/api
JWT_SECRET=<auto-generated-64-char-random>
DATABASE_URL=postgresql://postiz-user:postiz-password@postiz-postgres:5432/postiz-db-local
REDIS_URL=redis://postiz-redis:6379
BACKEND_INTERNAL_URL=http://localhost:3000
TEMPORAL_ADDRESS=temporal:7233
IS_GENERAL=true
STORAGE_PROVIDER=local
UPLOAD_DIRECTORY=/uploads
NEXT_PUBLIC_UPLOAD_DIRECTORY=/uploads
```

### Tier 2: Optional (User-configured)
Social media integrations - user edits `.env` or `postiz.env`:

```env
# Social Platforms
X_API_KEY=
X_API_SECRET=
LINKEDIN_CLIENT_ID=
LINKEDIN_CLIENT_SECRET=
# ... etc
```

### Tier 3: Advanced
OAuth, Stripe, Cloudflare R2 - power users only

## UX Flow

### First-Time Install

```
1. User clicks "Install"
2. Launcher:
   - Clones postiz-docker-compose
   - Generates .env with random JWT_SECRET
   - Pulls all docker images (shows progress)
   - Shows "Installation complete"
3. Menu changes to show "Start" button
```

### Starting the App

```
1. User clicks "Start"
2. Launcher:
   - Runs `docker compose up -d`
   - Waits for health checks
   - Shows status of each container
   - Opens browser to http://localhost:4007
3. Menu shows "Stop" and "Open" buttons
```

### Stopping the App

```
1. User clicks "Stop"
2. Launcher runs `docker compose down`
3. Containers stop but data persists
```

### Configuration

```
1. User clicks "Configure"
2. Launcher opens .env file in system editor
3. User edits and saves
4. Launcher prompts: "Restart to apply changes?"
```

## Docker Compose Strategy

**Option A: Clone upstream repo** (Recommended)
- Pros: Always in sync with official setup
- Cons: Dependency on external repo

**Option B: Embed compose file**
- Pros: Self-contained, can customize
- Cons: Must manually sync updates

**Decision:** Option A - Clone `gitroomhq/postiz-docker-compose`, but provide our own `.env` generation.

---

## Security Considerations

1. **JWT_SECRET**: Auto-generate 64-character random string
2. **Database passwords**: Use secure defaults (user can change)
3. **No secrets in repo**: `.env` is generated locally
4. **Volumes**: Persist data in Docker volumes

---

## Resource Requirements

- **Minimum:** 2GB RAM, 2 vCPUs (per Postiz docs)
- **Recommended:** 4GB RAM for smooth operation
- **Disk:** ~2GB for images + data

## Notes

- Temporal stack is required (not optional) - Postiz uses it for scheduled posts
- Elasticsearch needs specific memory settings (`ES_JAVA_OPTS=-Xms256m -Xmx256m`)
- First startup takes 1-2 minutes for all containers to be healthy
- User must have Docker installed (Pinokio requirement)
